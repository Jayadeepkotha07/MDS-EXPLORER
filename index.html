Understanding MDS

Multidimensional Scaling explained — from concept to algorithm, with a worked example and real-world applications.

① Concept Description

Definition

Multidimensional Scaling (MDS) is a dimensionality reduction technique that takes a matrix of pairwise distances (or dissimilarities) between objects and produces a low-dimensional coordinate layout that best preserves those distances.

Purpose

MDS enables humans to visually understand complex, high-dimensional relationships. When objects have known distances but unknown positions, MDS recovers a spatial arrangement that reflects their similarity structure.

Where It Is Used

Used in psychology (perceptual mapping), bioinformatics (gene expression), NLP (word embeddings), geography (map reconstruction), marketing (brand positioning), and social network analysis.

Step-by-Step Explanation

  1. 1

    Collect the Distance Matrix

    Start with an n×n symmetric matrix D where D[i][j] represents the dissimilarity between object i and object j. The diagonal is always 0 (each item is identical to itself).

  2. 2

    Initialize Random Coordinates

    Assign each of the n objects a random position in 2D (or 3D) space. These are our starting coordinates X, which will be iteratively improved.

  3. 3

    Compute Euclidean Distances in Layout

    Calculate d[i][j] = √Σ(X[i][k] − X[j][k])² — the actual distance between all pairs in the current layout space.

  4. 4

    Compute Stress (Error)

    Stress = √[ Σ(d[i][j] − δ[i][j])² / Σd[i][j]² ]. This measures how well the layout distances match the original dissimilarities. Lower stress = better fit.

  5. 5

    Compute Gradient & Update Positions

    Calculate the gradient of stress with respect to each coordinate. Move each point by a small step (learning rate) in the direction that reduces stress. Repeat from step 3.

  6. 6

    Center & Convergence Check

    Center the layout at the origin after each iteration. Stop when stress stops improving significantly or maximum iterations are reached. The final layout is the MDS solution.

Metric MDS

Preserves the actual magnitude of distances. If object A is twice as far from B as C is, the layout will reflect that ratio. Uses SMACOF / gradient descent optimization.

Non-Metric MDS

Only preserves the rank order of distances using isotonic regression (PAV algorithm). More flexible — works even when only ordinal relationships are known.

Stress Function

The objective being minimized: Stress = √[Σ(dᵢⱼ − δᵢⱼ)² / Σdᵢⱼ²]. Values near 0 indicate an excellent fit; above 0.2 suggests poor representation.

Gradient Descent

Each iteration nudges each point in the direction that reduces stress the most. The learning rate is adaptive: it shrinks when stress increases and grows when it decreases.

Using MDS Explorer

Everything you need to get started and make the most of the interactive features.

🚀 Getting Started

  • Navigate to the Playground page from the top nav.
  • Choose a preset dataset from the left panel (Cities, Fruits, Movies, Emotions).
  • Click ▶ Run MDS to start the animated optimization.
  • Watch the stress bar drop as the layout converges.

🔧 Controls

  • Toggle 2D / 3D view for different perspectives.
  • Switch Metric / Non-Metric mode to compare algorithms.
  • Drag the Iterations slider to control convergence depth.
  • Adjust Speed to slow down or speed up animation.

✏️ Custom Data

  • Select the Custom preset from the dataset list.
  • Enter labels — one per line (e.g., Apple, Banana, Cherry).
  • Enter the n×n distance matrix with values separated by commas.
  • Click Apply and then Run MDS to visualize.

🌐 3D Navigation

  • Click + Drag on the 3D plot to rotate the scene.
  • Scroll to zoom in and out on the 3D layout.
  • Enable Auto Rotate for a continuous spin effect.
  • Enable Galaxy Mode to add a starfield background.

⌨️ Keyboard Shortcuts

  • Press R to re-run MDS from scratch.
  • Press Esc to exit Add Point mode.
  • Click any point to see its coordinates and nearest neighbors.

📤 Export

  • Click ↓ Export in the playground to download the visualization as a PNG image.
  • In 2D mode, a high-resolution 1400×900 image is generated.
  • In 3D mode, the current canvas frame is captured.

Frequently Asked Questions

What does "Stress" mean in MDS? ▾
Stress is a normalized measure of how well the 2D/3D layout preserves the original pairwise distances. A stress of 0 means a perfect fit — all distances are exactly preserved. Values below 0.05 are excellent, below 0.1 are good, and above 0.2 indicate that the layout is a poor representation of the original data.
What is the difference between Metric and Non-Metric MDS? ▾
Metric MDS preserves the actual numerical values of distances in the layout — so if object A is twice as far from B as from C, the layout will reflect that ratio. Non-Metric MDS only preserves the rank ordering (A is farther from B than from C) using isotonic regression, making it more robust to non-linear or ordinal data.
Why do I get different results each time I run MDS? ▾
MDS starts with random initial positions and uses gradient descent to optimize. Because the starting point is random, different runs may converge to different local minima that are rotations, reflections, or slight variations of each other. The stress value indicates which solution fits better — lower is always better.
What format should my custom distance matrix be in? ▾
Enter an n×n symmetric matrix where each row is comma-separated numbers and each row is on a new line. The diagonal must be 0 (each item has zero distance from itself) and the matrix must be symmetric: D[i][j] = D[j][i]. The matrix size must match the number of labels you entered.
